# Prompt Template

Use this template when the user wants a ready-to-run prompt for one supported site.

```text
请直接完成，不要停下来询问我，不要展示抓取过程、命令过程或推理过程。你的任务是基于当前 opencli 工程，生成一份“今日 {站点名} 高浓度简报”。

执行原则：
- 优先复用当前工程已有的 opencli 命令与轻量 public API 能力
- 只在当前命令缺字段时，才允许做轻量补充
- 轻量补充仅限：
  1. 同一公开 API 的缺失字段补全
  2. 当前工程内极小的 repo-local YAML / TS extractor
- 禁止使用 Playwright、浏览器录制、重型自动化方案
- 如果个别条目抓取失败，直接跳过并补位
- 若样本量不足，不要伪造，按真实数据继续

任务流程：

1. 先抓取 {站点名} 的广义候选池，目标 {样本数} 条左右。
2. 只基于标题和轻量元信息做第一轮筛选，过滤水贴、情绪贴、无信息增量贴，保留 15-20 条最值得深读的内容。
3. 对这 15-20 条抓取详情与评论 / 回复。
4. 评论区去噪，只保留可执行建议、亲测经验、纠偏意见、最佳实践、失败案例和高质量分歧。
5. 按主题分类输出，排序依据是：
   信息密度 × 实用性 × 讨论质量

每条内容严格使用以下格式：

[原标题 + 紧凑链接]

核心脉络：
用 1-2 句话概括真正问题、坑点、方案对比或干货。

评论区风向：
提炼最有价值的解决方案、经验、共识或分歧。

✅ 一句话结论：
给出最值得带走的可执行判断。

全部条目输出完后，再补：
- 今日总体风向（3-5 条）
- 今日最值得立即试试的 5 个点

输出要求：
- 全程中文
- 高浓度、少废话
- 不展示抓取过程
- 不展示推理过程
- 不灌水
```
